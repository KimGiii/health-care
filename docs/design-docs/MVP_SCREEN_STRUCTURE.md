# MVP 화면 구조
## 운동·식단·신체변화 기반 목표 달성 서비스

**버전:** 1.1  
**작성일:** 2026-04-10  
**목적:** MVP에서 필요한 화면을 정의하고, 각 화면의 책임과 핵심 UI 블록을 정리한다.

---

## 1. 화면 설계 원칙

- 사용자는 `기록`보다 `지속`에 더 큰 어려움을 느낀다.
- 따라서 화면은 입력 속도, 현재 상태 파악, 목표 대비 변화 체감에 초점을 둔다.
- MVP는 기능 확장보다 `매일 기록 루프`를 완성하는 데 집중한다.

핵심 루프:

`목표 설정 → 운동/식단 기록 → 신체 변화 기록 → 주간 요약 확인 → 다음 행동 결정`

## 3. 화면별 정의

### 3.1 온보딩

**목적**
- 서비스 가치와 기록 흐름을 빠르게 이해시킨다.

**핵심 요소**
- 서비스 소개 슬라이드 3장
- 핵심 가치 메시지
- 시작하기 버튼
- 로그인 / 회원가입 진입

**표시 정보**
- 운동, 식단, 몸 변화를 한 곳에 기록
- 목표 대비 변화 시각화
- 주간 회고와 피드백

---

### 3.2 회원가입 / 로그인

**목적**
- 계정 생성 및 진입 장벽 최소화

**핵심 요소**
- 이메일/비밀번호 로그인
- 약관 동의
- 기본 프로필 입력

> 소셜 로그인(Apple ID, Google)은 MVP 제외. 이메일/비밀번호 인증만 지원한다.

**입력 항목**
- 이메일
- 비밀번호
- 닉네임

---

### 3.3 목표 설정

**목적**
- 사용자가 어떤 변화를 원하는지 명확히 설정한다.

**핵심 요소**
- 목표 유형 선택
- 목표 기간 설정
- 시작 상태 입력
- 목표 수치 입력

**입력 항목**
- 목표 유형: 감량 / 증량 / 유지 / 체형 개선
- 목표 시작일
- 목표 종료일
- 시작 체중
- 목표 체중
- 보조 목표: 주당 운동 횟수, 단백질 섭취 목표

**출력 요소**
- 예상 진행률 카드
- 이번 주 해야 할 기본 액션

---

### 3.4 홈 대시보드

**목적**
- 오늘 해야 할 행동과 현재 진행 상태를 한눈에 보여준다.

**핵심 요소**
- 오늘의 목표 요약
- 기록 CTA
- 주간 진행률
- 최근 변화 하이라이트

**주요 카드**
- 오늘 운동 기록 여부
- 오늘 식단 기록 여부
- 이번 주 체중/치수 변화
- 목표 대비 진행률
- 연속 기록 일수
- 개인 기록(PR) 달성 시 PR 알림 카드 (당일 달성한 경우에만 표시)

**핵심 액션**
- 운동 기록하기
- 식단 기록하기
- 몸 변화 기록하기
- 주간 회고 보기

---

### 3.5 기록 허브

**목적**
- 오늘 남길 수 있는 기록 유형을 빠르게 선택하게 한다.

**핵심 요소**
- 기록 유형 선택 카드
- 최근 기록 불러오기
- 템플릿 진입

**기록 유형**
- 운동
- 식단
- 체중/체지방/치수
- 사진
- 메모

---

### 3.6 운동 기록 화면

**목적**
- 운동 세션을 빠르게 저장한다.

**핵심 요소**
- 운동 종류 선택
- 세트/반복/중량 입력
- 이전 기록 불러오기
- 메모

**입력 블록**
- 날짜
- 운동명
- 세트 목록
- 반복 횟수
- 중량
- 운동 시간
- 세션 메모

**보조 기능**
- 최근 루틴 불러오기
- 세트 입력 시 이전 기록 및 개인 최고 기록(PR) 인라인 표시
- 저장 후 요약 보기 (PR 달성 시 축하 메시지 포함)

---

### 3.7 식단 기록 화면

**목적**
- 식사 기록을 부담 없이 빠르게 남긴다.

**핵심 요소**
- 식사 유형 선택
- 음식명/사진 입력
- 칼로리와 탄단지 입력
- 자주 먹는 음식 템플릿

**입력 블록**
- 식사 시간
- 음식명
- 섭취량
- 칼로리
- 탄수화물
- 단백질
- 지방
- 사진
- 메모

**보조 기능**
- 최근 식사 불러오기
- 반복 식단 저장

> 바코드 스캔은 MVP 제외. 음식명 텍스트 검색 기반 입력만 지원한다.

---

### 3.8 신체 변화 기록 화면

**목적**
- 몸 변화의 핵심 수치를 누적 기록한다.

**핵심 요소**
- 체중 입력
- 체지방률 입력
- 둘레 입력
- 전신 사진 업로드

**입력 블록**
- 체중
- 체지방률
- 골격근량
- 허리/가슴/엉덩이/허벅지 둘레
- 정면/측면 사진
- 컨디션 메모

---

### 3.9 변화 분석 화면

**목적**
- 기록 데이터를 목표 관점에서 해석해 보여준다.

**핵심 요소**
- 목표 진행률 그래프
- 체중/체지방/치수 추이
- 운동 빈도 및 볼륨 추이
- 식단 목표 달성률

**주요 섹션**
- 이번 주 변화 요약
- 지난 4주 추세
- 가장 잘 지켜진 습관
- 부족한 행동 신호

---

### 3.10 주간 회고 화면

**목적**
- 사용자가 한 주를 돌아보고 다음 행동을 결정하게 한다.

**핵심 요소**
- 이번 주 성과 요약
- 잘한 점 / 어려웠던 점 입력
- 다음 주 액션 설정

**입력/출력 블록**
- 이번 주 운동 횟수
- 식단 준수율
- 몸 변화 요약
- 자유 회고 메모
- 다음 주 목표 1~3개

---

### 3.11 히스토리 / 캘린더

**목적**
- 기록 누적 히스토리를 날짜 단위로 탐색한다.

**핵심 요소**
- 월간 캘린더
- 날짜별 기록 요약
- 특정 날짜 상세 보기

**표시 항목**
- 운동 여부
- 식단 여부
- 신체 변화 기록 여부
- 회고 작성 여부

---

### 3.12 마이페이지 / 설정

**목적**
- 사용자 정보, 목표, 알림, 개인정보 관련 설정을 관리한다.

**핵심 요소**
- 프로필
- 현재 목표
- 알림 설정
- 데이터 내보내기
- 계정 관리

**알림 설정 상세** (항목별 개별 토글)
- 기록 완료 후 목표 대비 현황 요약
- 개인 기록(PR) 달성 알림
- 주간 회고 작성 유도 (매주 일요일 저녁)
- 미기록 리마인더 (설정한 시간에 오늘 기록이 없을 경우)

---

## 4. MVP 우선순위

### P0

- 회원가입 / 로그인 (이메일/비밀번호)
- 목표 설정
- 홈 대시보드 (PR 카드 포함)
- 운동 기록 (PR 감지 및 표시 포함)
- 식단 기록 (텍스트 검색 입력)
- 신체 변화 기록
- 변화 분석 기본 화면
- 알림 설정 (4종 항목별 토글)

### P1

- 주간 회고
- 히스토리 / 캘린더
- 반복 입력 템플릿

### P2

- 사진 비교
- 자동 인사이트
- 추천 액션
- 공유 기능

---

## 5. MVP 권장 탭 구조

하단 탭 기준 5개를 권장한다.

1. 홈
2. 기록
3. 변화
4. 회고
5. 마이

이 구조의 장점:

- 사용 목적이 명확하다.
- 운동/식단/몸 변화가 `기록`으로 묶여 복잡도가 줄어든다.
- 사용자가 자주 보는 요약 화면과 입력 화면이 분리된다.

## 6. 첫 출시 기준 핵심 화면 수

출시 최소 구성은 8개 화면이면 충분하다.

1. 온보딩
2. 로그인
3. 목표 설정
4. 홈
5. 기록 허브
6. 운동 기록
7. 식단 기록
8. 신체 변화 기록

추가 권장 화면:

9. 변화 분석
10. 주간 회고

---

## 7. 핵심 성공 조건

**UX 기준 (화면 설계 수준)**
- 첫 기록까지 3분 이내
- 하루 기록 완료까지 5분 이내
- 홈 화면에서 현재 목표 상태를 즉시 이해 가능
- 주간 단위 변화가 숫자와 시각 자료로 동시에 보일 것

**제품 지표 기준 (PRD 7.3 기준)**

| 지표 | 목표 |
|------|------|
| 목표 설정 완료율 | 70% 이상 |
| 첫 주 3회 기록 달성 비율 | 40% 이상 |
| 주간 회고 작성 비율 | 25% 이상 |
| 4주차 유지율 | 30% 이상 |
| 기록 후 대시보드 재방문율 | 60% 이상 |
