# 사용자 시나리오 / 와이어프레임
## 운동·식단·신체변화 기반 목표 달성 서비스

**버전:** 1.1  
**작성일:** 2026-04-10  
**목적:** MVP 기준 대표 사용자 시나리오와 저해상도 와이어프레임을 정의한다.

---

## 1. 대표 사용자 시나리오

### 시나리오 A. 첫 사용자 온보딩

**상황**
- 사용자는 체중 감량 또는 체형 개선 목표를 가지고 처음 앱에 들어온다.

**목표**
- 서비스의 핵심 가치를 이해하고, 첫 목표를 설정한다.

**흐름**
1. 온보딩 화면에서 서비스 소개를 확인한다.
2. 이메일/비밀번호로 회원가입 또는 로그인을 진행한다. (소셜 로그인 미지원)
3. 현재 신체 정보와 목표를 입력한다.
4. 홈 대시보드에서 첫 기록 가이드를 본다.
5. 첫 운동 또는 첫 식단 기록을 시작한다.

**검증 포인트**
- 목표 설정까지 이탈 없이 도달하는가
- 목표 입력 과정이 복잡하지 않은가
- 첫 기록 행동으로 자연스럽게 이어지는가

---

### 시나리오 B. 오늘 운동 기록하기

**상황**
- 사용자는 운동을 마치고 오늘 운동 세션을 기록하려 한다.

**목표**
- 1분 내 운동 기록 저장

**흐름**
1. 홈 또는 기록 허브에서 `운동 기록`을 선택한다.
2. 운동 종류를 고른다.
3. 세트/반복/중량을 입력한다. (세트 입력 시 이전 기록과 PR이 인라인으로 표시된다)
4. 저장 버튼을 누른다.
5. 저장 후 오늘 운동 요약이 표시된다.
6. PR 달성 시 축하 메시지와 함께 홈 대시보드에 PR 카드가 노출된다.

**검증 포인트**
- 운동 입력 단계 수가 과도하지 않은가
- 사용자가 자주 하는 운동을 재입력 없이 불러올 수 있는가
- 저장 후 보상(PR 달성 피드백)이 충분한가

---

### 시나리오 C. 오늘 식단 기록하기

**상황**
- 사용자는 식사 직후 식단을 빠르게 남기고 싶다.

**목표**
- 30초~1분 내 식단 기록 완료

**흐름**
1. 기록 허브에서 `식단 기록`을 선택한다.
2. 아침/점심/저녁/간식을 선택한다.
3. 음식명 텍스트 검색으로 음식을 찾거나 직접 입력한다. (바코드 스캔 미지원)
4. 칼로리/탄단지를 확인하고 섭취량을 조정한다. 사진·메모로만 기록도 가능하다.
5. 저장한다.
6. 남은 칼로리/단백질 목표를 바로 확인한다.

**검증 포인트**
- 텍스트 검색만으로도 입력이 충분히 빠른가
- 사진과 메모만으로도 최소 기록이 가능한가
- 저장 후 즉시 의미 있는 피드백을 받는가

---

### 시나리오 D. 신체 변화 기록하기

**상황**
- 사용자는 주 1회 몸 변화를 확인하고 싶다.

**목표**
- 체중, 치수, 사진을 일관되게 누적 기록

**흐름**
1. 홈 또는 기록 허브에서 `몸 변화 기록`을 선택한다.
2. 체중과 체지방률을 입력한다.
3. 허리 등 주요 둘레를 입력한다.
4. 전신 사진을 업로드한다.
5. 저장 후 지난 기록과 비교된 요약을 확인한다.

**검증 포인트**
- 한 번에 너무 많은 수치를 요구하지 않는가
- 최소 입력과 확장 입력이 구분되는가
- 사진 기록에 심리적 부담이 크지 않은가

---

### 시나리오 E. 주간 회고 확인하기

**상황**
- 사용자는 일주일의 변화를 돌아보고 다음 주 액션을 정리하려 한다.

**목표**
- 데이터를 행동 계획으로 전환

**흐름**
1. 홈에서 `주간 회고` 카드를 선택한다.
2. 이번 주 운동 횟수, 식단 준수율, 체중 변화 요약을 본다.
3. 잘한 점과 어려웠던 점을 입력한다.
4. 다음 주 목표 1~3개를 선택한다.
5. 저장 후 다음 주 홈 화면에 반영된다.

**검증 포인트**
- 숫자만 나열되지 않고 해석 가능한가
- 회고 작성이 너무 길지 않은가
- 다음 행동이 명확히 정리되는가

---

### 시나리오 F. 알림을 받고 앱으로 재진입하기

**상황**
- 사용자가 운동 기록을 저장한 뒤 앱을 닫았다. 잠시 후 "오늘 벤치프레스 PR 달성!" 푸시 알림이 도착한다.

**목표**
- 알림이 재방문을 유도하고, 홈에서 성과를 즉시 확인할 수 있다.

**흐름**
1. 푸시 알림을 탭한다.
2. 홈 대시보드로 바로 이동하며, PR 달성 카드가 상단에 강조 표시된다.
3. 카드를 탭하면 해당 운동 세션 요약이 열린다.
4. 이전 기록과 이번 기록을 비교해 확인한다.
5. 홈으로 돌아와 오늘 남은 기록(식단 등)을 확인한다.

**검증 포인트**
- 알림 탭 → 해당 화면 진입이 1초 이내인가
- PR 카드가 즉각적으로 동기와 보람을 주는가
- 홈 화면에서 다음 행동을 자연스럽게 이어가게 되는가

## 2. 주요 사용자 흐름

### Flow 1. 첫 진입

`온보딩 → 회원가입(이메일/비밀번호) → 목표 설정 → 홈 → 첫 기록`

### Flow 2. 일일 사용

`홈 → 기록 허브 → 운동/식단 입력 → 저장 → (PR 달성 시 축하 팝업) → 홈 복귀`

### Flow 3. 주간 사용

`홈 → 몸 변화 기록 → 변화 분석 → 주간 회고`

### Flow 4. 알림 재진입

`푸시 알림 탭 → 홈(PR 카드 강조) → 세션 요약 확인 → 다음 기록 행동`

## 4. MVP 검증용 핵심 태스크

사용자 테스트에서는 아래 태스크를 반드시 수행하게 해야 한다.

1. 목표를 설정해보기
2. 운동 기록 1건 입력하기
3. 식단 기록 1건 입력하기
4. 몸 변화 기록 1건 입력하기
5. 지난 기록을 보고 현재 상태 설명하기
6. 주간 회고를 작성해보기

## 5. 테스트 시 꼭 확인할 질문

- 어떤 화면에서 가장 오래 멈췄는가
- 어떤 입력 항목이 가장 번거로웠는가
- 저장 후 얻는 보상이 충분했는가
- 홈 화면만 보고 지금 상태를 이해할 수 있었는가
- 이 앱을 매일 쓸 가능성이 얼마나 있는가
