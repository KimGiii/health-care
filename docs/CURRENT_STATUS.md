## Phase별 상세 상태

### Phase 0: 환경 구축

상태: 완료

- `backend/build.gradle.kts`, `docker-compose.yml`, `application.yml`, Flyway 마이그레이션 구성 완료
- PostgreSQL, Redis, LocalStack 기반 로컬 구성이 존재

### Phase 1: 인증 & 사용자

상태: 기능 구현 완료, 세부 정책 정리 단계

- 인증/프로필 기본 CRUD는 동작 가능한 수준
- JWT 기반 보안 체인은 구현됨
- 회원가입에서 초기 프로필 정보와 지역 설정 저장 가능
- `goalType`은 현재 목표 API에서 관리하도록 분리 유지
- 관련 테스트는 서비스/컨트롤러 관점에서 더 보강 필요

### Phase 2: 운동 기록

상태: 완료

- 카탈로그, 세션 생성/조회/삭제 흐름이 구현되어 있음
- 도메인 구조와 테스트도 비교적 안정적

### Phase 3: 식단 기록

상태: 거의 완료, 테스트 보정 필요

- 식단 기록 CRUD와 식품 검색은 구현됨
- 외부 공공데이터 API 연동도 포함됨
- 현재 테스트 1건 실패로 인해 안정화 마무리가 필요

### Phase 4: 신체 측정 & 진행 사진

상태: 핵심 MVP 기능 구현 완료, 후속 처리 남음

- 신체 측정 기록 API는 이미 구현됨
- 진행 사진 업로드는 presigned URL 기반 MVP로 구현됨
- Terraform 기준 AWS S3 버킷/IAM 정책 골격이 추가됨
- EXIF 제거, 썸네일 생성, 업로드 후처리는 다음 단계 과제

### Phase 5: 목표 & 인사이트

상태: 기초 CRUD만 구현

- 목표 생성/조회/수정/포기는 구현됨
- 목표 진행률 계산, 체크포인트, 인사이트 생성은 아직 미완성

### Phase 6: MVP 출시 준비

상태: 미착수

- E2E 테스트
- 개인정보 처리방침
- 배포/릴리즈 준비
